🔄 What is Asynchronous Code in JavaScript? JavaScript is single-threaded, meaning it can do one thing at a time. But thanks to asynchronous programming, JavaScript can handle tasks like network requests, timers, or file I/O without blocking the main thread. 🧠 Key Concepts **Callback**: A function passed as an argument to another function, which is executed after the first function completes. Synchronous: Code Executes line by line. Each step waits for the previous one to finish. Asynchronous: Code Doesn’t wait. Starts a task and moves on. The result comes later via a callback, promise, or async/await. Event Loop: The system that handles async code in JS. It moves completed async tasks back into the call stack to be executed. ┌────────────────────┐ │ JS Call Stack │ └────────┬───────────┘ │ ▼ ┌──────────────────────┐ │ Web APIs / Node APIs│ <---- setTimeout, fetch, etc. └────────┬─────────────┘ │ ┌─────────▼─────────────┐ │ Callback / Task Queue│ └─────────┬─────────────┘ │ (via Event Loop) ▼ ┌───────────────┐ │ Call Stack │ <--- Executes callback now └───────────────┘
